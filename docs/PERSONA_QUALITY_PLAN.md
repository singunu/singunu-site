## 이슈 1 — 답변에 `**볼드**` 등 마크다운 AI 티

**증상**: 챗 답변에 `**강조**`, 리스트 마커, 헤더가 그대로 노출 — 프론트는 plain text 렌더링이라 별표가 문자로 보임. "사람이 채팅 치는 느낌"이 깨진다.

**해결 (2단 방어, 🤖)**:
1. **프롬프트 규칙 추가** — `PersonaService.buildSystemPrompt`의 지침 블록에:
   - 마크다운 문법(볼드/이탤릭/헤더/리스트/코드블록) 금지, 순수 문장으로만
   - 메신저 대화 톤: 짧은 문단, 한 답변 2~4문장 기본 (길어야 할 때만 예외)
   - 이모지·과한 감탄 절제 (보이스팩 톤과 일치)
2. **서버 후처리 (보강)** — 스트리밍 delta에서 마크다운 토큰 스트립:
   - `**`, `__`, `` ` ``, 행머리 `#`/`- `/`1. ` 정도만 정규식 제거 (과잉 변환 금지)
   - 주의: delta 경계에 `*`가 걸칠 수 있음 → ChatService의 폴백 토큰 prefix 버퍼처럼 **1~2자 lookahead 버퍼** 필요
   - 대안(간단): 프론트에서 렌더 직전 스트립 — 서버 수정 없이 `ChatApp.tsx`에서 처리. 1차는 프론트 스트립+프롬프트로 충분할 수 있음. 서버 후처리는 로그 정합성(로그에도 깨끗한 답 저장)이 필요할 때.

**검증**: "네 장점 3가지 알려줘" 같은 리스트 유도 질문 → 별표/번호 없이 문장으로 나오는지.

## 이슈 2 — 먹거리·카페·음식점 취향 데이터 없음

**증상**: "뭐 먹는 거 좋아해?", "단골 카페 있어?" → 데이터가 없어 얕은 답변 또는 폴백.

**해결**:
1. 🙋 **볼트(`me/`)에 원본 작성** — 항목 제안:
   - 좋아하는 음식 / 싫어하거나 못 먹는 것 / 최애 야식
   - 커피·차 취향 (원두, 메뉴, 홈카페 여부)
   - 단골 카페·음식점 — **⚠ 공개 수위 결정 필요**: 실제 상호·동네는 위치 노출 리스크. 권장: "동네 조용한 로스터리" 수준으로 익명화하거나 프랜차이즈/유명 맛집만 실명
   - 술 (주종·주량·술자리 스타일), 디저트, 요리 하는지
   - 음식과 얽힌 에피소드 1~2개 (답변에 생기를 주는 재료)
2. 🤖 볼트 원본 → `public_persona.md` `## 취향` 섹션에 큐레이션 (REVIEW_CHECKLIST 통과)
3. 🙋 최종 검수 → 🤖 서버 반영 (아래 "반영 절차" 참고)

## 이슈 3 — 시점 의존 정보가 하드코딩됨

**증상**: "구직 중", 경력 연차, 나이, "요즘 목표" 등이 특정 시점 기준으로 박제 → 시간이 지나면 거짓말이 됨.

**해결 (구조 변경, 🤖 설계 + 🙋 내용)**:
1. `public_persona.md`를 **불변 정보와 시점 정보로 분리**:
   - 불변: 가치관, 성격, 걸어온 길(연도 명시 과거형), 취향(잘 안 변함)
   - `## [현재 상태 — 기준일: 2026-07-07]` 섹션 신설: 구직/재직 상태, 요즘 목표, 근황을 여기로 모음
2. **상대 표현 금지**: "작년", "요즘", "최근" → 절대 연도/기준일 기반 서술로 치환
3. **프롬프트 지침 추가**: "현재 상태 섹션의 기준일 이후는 알 수 없다 — 기준일 언급 + 최신 근황은 본체에게 직접(폴백 유도)"
   - 예: "2026년 7월 기준으로는 구직 중이었어요. 지금은 어떤지 본인에게 직접 물어보실래요?"
4. **업데이트 운영 규칙**: 상태 변화(취업 등) 시 + 분기 1회 `[현재 상태]` 갱신. TODO_LIST에 반복 항목으로 등록.

## 지속 개선 루프 — 대화 로그 기반 DB 다듬기

1. 🤖 MongoDB Atlas `conversations` 컬렉션에서 주기적으로 추출:
   - guardFlag=FALLBACK/INPUT_BLOCKED 질문 (못 답한 것)
   - 답변이 어색했을 법한 짧은/긴 응답 샘플
2. 🙋 훑어보고 "답할 수 있어야 하는 질문" 선별 → 볼트에 원본 추가
3. 🤖 큐레이션 반영. **예상 질문 갭 체크리스트**로 선제 보강:
   - 친구 관점: 연애관, 주말에 뭐함, 운동, 게임, 음악, 여행, MBTI 심화
   - 채용담당자 관점: 강점/약점, 협업 스타일, 이직 사유, 연봉(→폴백 유도가 정답), 포트폴리오
4. 톤 문제 체크: 첫인사 반복, 존댓말 일관성, 답변 길이 편차
